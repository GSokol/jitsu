# Jitsu Configurator

`@jitsucom/configurator` image should be used for Jitsu Configurator deployment

## Exposed ports

`@jitsucom/configurator` image exposes `8000` port

## Environment Variable

<APIParam small={true}  name="UI_AUTH_ACCESS_SECRET" dataType="string" required={true} type="Docker Env Var">
    Secrets used to sign and refresh JWT Access Token. Use any random string.{' '}
    Feel free to rotate them, change in the settings
    will force logout of of Configurator
</APIParam>
<APIParam small={true}  name="UI_AUTH_REFRESH_SECRET" dataType="string" required={true} type="Docker Env Var">
    Secrets used to sign and refresh JWT Access Token. Use any random string.{' '}
    Feel free to rotate them, change in the settings
    will force logout of of Configurator
</APIParam>
<APIParam small={true}  name="CONFIGURATOR_ADMIN_TOKEN" dataType="string" required={true} type="Docker Env Var">
    Auth token for authorizing requests from Jitsu Server. Should match one configured in <a href="/docs/deployment/deploy-with-docker/jitsu-server">Jitsu Server</a>
</APIParam>
<APIParam small={true}  name="JITSU_SERVER_URL" dataType="string" required={true} type="Docker Env Var">
    URL of Jitsu Server
</APIParam>
<APIParam small={true}  name="SERVER_ADMIN_TOKEN" dataType="string" required={true} type="Docker Env Var">
    Auth token for authorizing requests to Jitsu Server. Should match one configured in <a href="/docs/deployment/deploy-with-docker/jitsu-server">Jitsu Server</a>
</APIParam>
<APIParam small={true}  name="REDIS_URL" dataType="string" required={true} type="Docker Env Var">
    Redis connection string: <code inline={true}>redis://:password@host:port</code> or{' '}
    <code inline={true}>redis://host:port without password</code>
</APIParam>
<APIParam small={true} name="TLS_SKIP_VERIFY" dataType="boolean" required={false} type="Docker Env Var">
    Redis: skip client certificate verification
</APIParam>
